Title - £ character in GRA digitization

funderburkjim :

A user submitted a correction under 'id' for Grassman. It appears (at bottom of a long entry) in displays as:
```
néd = ná£íd siehe für sich.
```
This is at bottom of page 206 of scans:
![image](https://cloud.githubusercontent.com/assets/6393033/5497605/53e26700-86e1-11e4-9feb-457c832bab19.png)

The user's suggestion was
```
best would be „ná‿íd“ possible is also „ná_íd“
```
Since I couldn't find the unicode representation for the first option I chose to use the 2nd option which is a simple ascii underscore character.

It happens that £ occurs many times (1000+) in the gra.txt digitization. A small sample suggests that it always occurs as a separator between vowels in hiatus. Since the underscore character does NOT occur in gra.txt it is safe to replace £ with _. At least in Emacs that curved underscore does not show up well (also I don't know what Unicode character it is) so it seems best to use the underscore.

Comments?
